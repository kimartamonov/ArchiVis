## Оценка рисков

### Топ-3 рисков по совокупности вероятности и влияния

#### Риск 1: API Architeezy не предоставляет нужных данных

| Параметр | Значение |
|----------|----------|
| Вероятность | Средняя (30-50%) |
| Влияние | Критическое — без данных нет продукта |
| Категория | Интеграция |
| Владелец | Tech lead |
| Ранний сигнал | Spike показывает, что нет endpoint для relationships или diagram membership |
| Мера реагирования | Fallback на demo-only MVP. Искать альтернативные источники (ArchiMate Open Exchange file import). Обратиться к Architeezy за помощью |
| Plan B | MVP работает только на demo dataset. Real integration — Phase 2 после получения доступа/документации |

#### Риск 2: Большие графы тормозят UI

| Параметр | Значение |
|----------|----------|
| Вероятность | Низкая-средняя (20-30%) |
| Влияние | Среднее — UX деградирует, но не ломается |
| Категория | Производительность / UX |
| Владелец | Разработчик |
| Ранний сигнал | React Flow тормозит при > 300 узлов. Layout занимает > 3 сек |
| Мера реагирования | Ограничить отображаемые узлы (только видимые слои). Упростить layout. Убрать анимации. Использовать canvas renderer вместо SVG |
| Plan B | Ограничить MVP до 300 элементов. Для больших моделей — только таблица |

#### Риск 3: Метрики воспринимаются как «магия» или дают нерелевантные результаты

| Параметр | Значение |
|----------|----------|
| Вероятность | Средняя (30-40%) |
| Влияние | Среднее — потеря доверия, продукт не принимается |
| Категория | UX / доверие |
| Владелец | Product owner |
| Ранний сигнал | Первый пользователь говорит «не понимаю, что значит этот score» |
| Мера реагирования | Добавить tooltip к каждой метрике. Показать «почему» (explain mode). Убрать композитные scores, оставить простые (degree, count). Получить feedback от архитектора |

### Дополнительные риски

#### Риск 4: CORS блокирует прямые запросы к Architeezy

| Параметр | Значение |
|----------|----------|
| Вероятность | Высокая (60-70%) |
| Влияние | Низкое — решаемо |
| Мера реагирования | Документировать запуск на localhost. Предложить простой CORS proxy (3 строки на Express). Попросить Architeezy добавить CORS headers |

#### Риск 5: Нет доступа к реальной модели для тестирования

| Параметр | Значение |
|----------|----------|
| Вероятность | Средняя (40%) |
| Влияние | Среднее — невозможно валидировать на реальных данных |
| Мера реагирования | Demo dataset покрывает все тестовые сценарии. Найти open source ArchiMate-модель для тестирования |

#### Риск 6: Один разработчик = bus factor 1

| Параметр | Значение |
|----------|----------|
| Вероятность | Низкая |
| Влияние | Критическое |
| Мера реагирования | Хорошая документация, чистый код, README, тесты. Привлечь второго contributor как можно раньше |

## Следующая волна вопросов после MVP

Если MVP успешен, возникнут:
1. **Масштаб:** Как работать с моделями > 1000 элементов? (Web Workers, агрегация, lazy loading)
2. **Persistence:** Нужно ли сохранять результаты анализа? (Backend, DB)
3. **Collaboration:** Как делиться результатами? (Shareable links, export improvements)
4. **Extensibility:** Как сообщество добавляет свои метрики? (Plugin API)
5. **Multi-source:** Как подключить Archi, TOGAF repos? (Connector SDK)
6. **Governance:** Как организовать open source проект? (Contributing guide, release process)
