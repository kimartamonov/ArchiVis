# 06_Nonfunctional_Requirements.md: вопросы к заполнению

## Каких данных не хватает в Idea-Final

- Почти отсутствуют измеримые нефункциональные требования.
- Не зафиксированы целевые пределы производительности, надежности, совместимости и безопасности.
- Нет критериев качества UX, объяснимости и расширяемости, хотя для идеи они критичны.

## Вопросы

- Какой размер модели должен считаться целевым для MVP: число элементов, отношений, диаграмм?
- За какое время допустимо загрузить модель и построить граф в целевом сценарии?
- За какое время пользователь должен получать результат impact analysis после выбора элемента?
- Какая задержка допустима для фильтрации, поиска и навигации по графу?
- Какие браузеры и платформы должны официально поддерживаться в MVP?
- Нужна ли адаптация под мобильные устройства или достаточно desktop-only режима?
- Какой уровень устойчивости обязателен: что продукт должен делать при временной недоступности API или частичной ошибке загрузки?
- Требуется ли кэширование данных, и если да, то какие ограничения по актуальности кэша допустимы?
- Какие требования по безопасности уже известны: тип аутентификации, хранение токенов, защита чувствительных данных, разграничение доступа?
- Нужно ли поддерживать работу с конфиденциальными архитектурными моделями без передачи данных во внешние сервисы?
- Какие требования к логированию и диагностике нужны для MVP?
- Нужны ли требования к объяснимости метрик как отдельное нефункциональное качество?
- Какой уровень доступности/Accessibility нужен для первой версии?
- Насколько важна расширяемость API/плагинов уже в архитектуре MVP, даже если сами плагины позже?
- Какие требования к сопровождаемости: покрытие тестами, модульность, документация, кодстайл, API-стабильность?
- Есть ли требования к лицензии, юридической чистоте зависимостей и open source governance, которые ограничивают архитектуру?
