# 03_Stakeholders_and_Users.md: вопросы к заполнению

## Каких данных не хватает в Idea-Final

- Есть список ролей, но не выделен один основной пользователь и не описан его рабочий контекст.
- Нет портретов пользователей: их зрелости, частоты использования, ограничений, мотивации и критериев успеха.
- Не определено, кто реально принимает результат MVP и чьи интересы конфликтуют между собой.

## Вопросы

- Кто является первичным пользователем MVP, ради которого нужно оптимизировать первый UX?
- Какая роль является вторичной, но критичной для принятия решения о ценности продукта?
- В каком типе организации и в каком контексте эти пользователи работают: enterprise, consultancy, внутренний архитектурный офис, команда трансформации?
- Каков типичный размер и зрелость репозитория у целевого пользователя?
- Как часто основной пользователь будет обращаться к продукту: ежедневно, еженедельно, перед review, по запросу изменений?
- Какой уровень ArchiMate- и архитектурной экспертизы у основного пользователя предполагается по умолчанию?
- Какие главные боли у каждого ключевого стейкхолдера: архитектора, владельца репозитория, менеджера трансформации, руководителя?
- Какие решения основной пользователь принимает на основе результатов анализа?
- Насколько для него важна объяснимость метрик по сравнению со скоростью получения результата?
- Нужен ли продукт только архитекторам или часть результатов должна быть понятна неархитекторам?
- В каком виде вторичные пользователи будут потреблять результат: через UI, экспорт, скриншоты, таблицы, evidence pack?
- Какие ограничения есть у пользователей: дефицит времени, слабое знание модели, недоверие к "магическим" score, необходимость печатных артефактов?
- Кто принимает MVP на стороне пользователя: архитектор-лид, владелец репозитория, спонсор, конкурсное жюри?
- Кто может быть против продукта или воспринимать его как конкурента существующим инструментам?
- Какие пользовательские компромиссы допустимы в первой версии: меньше гибкости ради скорости, меньше визуальной красоты ради полезности, меньше покрываемых ролей ради четкого ICP?
