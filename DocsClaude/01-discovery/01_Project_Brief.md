# 01. Project Brief

> Главный вопрос документа: «Что это за продукт и зачем он вообще нужен?»

---

## Название проекта

**ArchiLens OSS**

## Слоган

**See impact, not just diagrams.**

---

## Краткая идея продукта

ArchiLens OSS — open source веб-приложение и аналитический слой поверх Architeezy API, которое превращает ArchiMate-модель из набора статических диаграмм в инструмент анализа влияния, критичности и качества архитектуры.

Продукт загружает ArchiMate-модель, собирает из неё единый граф элементов, отношений, представлений и слоёв, рассчитывает метрики связанности и покрытия и показывает архитектуру через сценарии анализа, а не через diagram rendering.

---

## Проблема, которую решает продукт

У пользователей ArchiMate повторяются одни и те же боли:

- Один и тот же объект размазан по разным диаграммам и views.
- Модель есть, но быстро понять последствия изменения конкретного элемента трудно.
- Критичные зависимости видны только экспертам, которые «и так всё знают».
- Модель постепенно деградирует: появляются элементы-сироты, слабое покрытие диаграммами, плохо связанные домены.
- Архитектурный репозиторий становится местом хранения знаний, но не инструментом ежедневных решений.

Хранить модель мало — нужно уметь **читать её как систему зависимостей, рисков и зон влияния**.

---

## Цель MVP

Подтвердить главную гипотезу:

> Архитектор, работающий с ArchiMate-моделью в Architeezy, может за 2-3 клика увидеть полную картину последствий изменения конкретного элемента — включая затронутые элементы, слои и диаграммы — и это реально экономит ему время по сравнению с ручным обходом диаграмм.

---

## Ожидаемый результат после MVP

Рабочий инструмент, который можно:

1. **Запустить локально** и подключить к реальному экземпляру Architeezy.
2. **Показать в демо** за 5 минут с понятной историей: выбрал элемент → увидел последствия → увидел диаграммы → выгрузил подграф.
3. **Опубликовать на GitHub** как open source проект с документацией, demo dataset и инструкцией для запуска.

---

## Заказчик и принятие решений

- **Инициатор и заказчик:** команда проекта (мы сами). Это независимый open source продукт.
- **Продуктовые решения:** product owner проекта.
- **Архитектурные и технические решения:** tech lead / ведущий разработчик.
- **Вторичный заказчик:** open source сообщество ArchiMate-практиков.
- **Стратегический партнёр:** Architeezy (первый источник данных, но не заказчик).

---

## Что считается успехом MVP

1. Пользователь подключается к Architeezy API, загружает модель и видит её как граф — за < 30 секунд.
2. По клику на элемент за < 3 секунды видит affected elements, слои и диаграммы на глубину до 3 шагов.
3. Orphan elements и low-coverage элементы определяются автоматически.
4. Подграф экспортируется в GraphML и CSV.
5. Демо-прогон по каноническому сценарию проходит успешно на модели от 50 элементов.

## Что считается неуспехом

- API Architeezy не предоставляет достаточно данных для построения графа связей.
- Impact analysis на реальной модели даёт нерелевантные результаты.
- Архитектор-эксперт при демо не видит ценности по сравнению с ручным обходом диаграмм.
- Построение графа на модели > 200 элементов занимает > 10 секунд.

## Ключевые ограничения на старте

| Ограничение | Значение |
|-------------|----------|
| Команда | 1-2 разработчика, product owner совмещает роль |
| Бюджет | Нулевой денежный бюджет. Только время участников |
| Сроки | 6 недель до рабочего MVP, первый демонстрируемый результат — через 2 недели |
| Доступ к API | Точные возможности API Architeezy станут понятны после spike-этапа |
| Лицензия | Только open source библиотеки (MIT / Apache 2.0 / BSD-совместимые) |
| Эксперты | Нужен хотя бы один человек с опытом ArchiMate для валидации результатов |

---

## Ключевые неясности на старте

1. Какие именно endpoints предоставляет API Architeezy (формат, аутентификация, доступные данные).
2. Поддерживает ли Architeezy CORS для внешних web-клиентов.
3. Есть ли доступ к реальной ArchiMate-модели для тестирования.

---

## Позиционирование

ArchiLens OSS — это **самостоятельный open source проект**, который начинает с интеграции с Architeezy как первого коннектора, но архитектурно не привязан к одному источнику данных. Это analysis layer для ArchiMate-моделей, а не плагин для конкретного продукта.

Продукт не пытается заменить Architeezy как систему управления репозиторием, ArchiMate-редактор, ручное моделирование или BI-систему. Он усиливает существующий репозиторий там, где пользователям важнее всего не редактирование, а **понимание и принятие решений**.

---

## Стартовый контекст для нового участника

Новому участнику достаточно знать:

1. **Что:** open source веб-инструмент для анализа ArchiMate-моделей — impact analysis, criticality, quality coverage.
2. **Чем отличается от viewer:** мы не показываем диаграммы. Мы строим единый граф модели и отвечаем на вопрос «что затронет изменение X».
3. **Откуда данные:** из Architeezy по API (read-only). В будущем — из других источников.
4. **Стек:** React + React Flow + elkjs + TypeScript. Графовый анализ — headless layer.
5. **Что важно сейчас:** MVP = impact analysis + coverage insights + export. Не редактор, не BI, не замена Architeezy.
6. **Как запустить:** `npm install && npm run dev`, подключить URL Architeezy и token.
