# 02. Scope and Boundaries

> Главный вопрос документа: «Что именно мы делаем в MVP, а что не делаем?»

---

## Что входит в MVP

### Обязательные сценарии (P0)

| # | Сценарий | Обоснование |
|---|----------|-------------|
| 1 | Подключение к Architeezy и загрузка модели | Без этого ничего не работает |
| 2 | Global Graph с фильтрами по слою и типу | Основной способ видеть модель как единое целое |
| 3 | Impact Analyzer — выбор элемента, affected elements/layers/diagrams на 1-2-3 шага | Главная ценность продукта |
| 4 | Таблица элементов с базовыми метриками | Альтернативный способ навигации |
| 5 | Coverage & Hygiene — orphan elements, low-coverage | Вторая ключевая ценность |
| 6 | Export в GraphML и CSV | Мост к внешним инструментам |

### Обязательные метрики (P0)

- `degree` (общее число связей)
- `in-degree` / `out-degree`
- `diagrams_count` (число диаграмм, где элемент присутствует)
- `orphan_flag` (элемент без связей или без диаграмм)

### Желательные метрики (P1)

- `impact_radius` (число элементов, затрагиваемых на глубину N шагов)
- `betweenness_centrality` (простая метрика центральности)

---

## Что сознательно НЕ входит в MVP

| Функция | Причина исключения |
|---------|-------------------|
| Criticality Radar как отдельный экран | Метрики centrality считаются, но отдельный экран — Phase 2 |
| Cross-Layer X-Ray | Красивая «вау»-фича, но сложная в реализации |
| PNG/SVG export визуализации | Скриншот пользователь сделает сам |
| JSON snapshot анализа | Nice-to-have |
| Plugin API / Extension API | Архитектурно заложим расширяемость, но plugin system — Phase 2+ |
| Saved views / shareable links | Требует persistence |
| Compare two states of model | Phase 2-3 |
| Мультимодельный анализ | Phase 3 |
| Уведомления / alerts по деградации | Phase 3 |
| Адаптеры к другим источникам | Post-MVP |
| CLI / batch mode | Post-MVP |
| Мобильный UI | Не нужен для целевого пользователя |
| Настройка quality rules пользователем | Phase 2 |

---

## Границы по масштабу и платформе

| Параметр | MVP-граница |
|----------|-------------|
| Моделей за раз | Одна |
| Источник данных | Только Architeezy API |
| Режим работы | Строго read-only, без записей в репозиторий |
| Целевой размер модели | До 500 элементов, до 1500 отношений |
| Платформа | Web, только desktop browsers (Chrome, Firefox, Edge) |
| Аутентификация | Нет собственной системы auth — пользователь указывает URL + token Architeezy |
| Persistence | Нет. Все данные in-memory. При закрытии вкладки — данные теряются |
| Глубина impact analysis | Фиксированные 1, 2, 3 шага |
| Язык интерфейса | Английский |

---

## Assumptions

1. API Architeezy предоставляет endpoints для получения элементов, отношений и диаграмм модели.
2. Аутентификация к API — через API token или Basic Auth.
3. ArchiMate-модели целевых пользователей содержат 100-500 элементов.
4. Пользователь имеет базовое знание ArchiMate (слои, типы элементов, отношения).
5. Данные модели не содержат конфиденциальной информации, запрещающей обработку в браузере.

---

## Зависимости

| Зависимость | Тип | Влияние |
|------------|-----|---------|
| Доступ к API Architeezy (credentials) | Внешняя | Блокирует spike. Без этого — только demo dataset |
| Формат API (endpoints, schema) | Внешняя | Определяет connector |
| CORS policy Architeezy | Внешняя | Может потребовать proxy |
| React Flow | Библиотека | Основа визуализации графа |
| elkjs | Библиотека | Auto-layout графа |

---

## Минимальный критерий полноты MVP

MVP считается завершённым, когда можно от начала до конца пройти канонический сценарий:

1. Подключиться к Architeezy (или загрузить demo dataset).
2. Увидеть модель как граф с цветовой кодировкой слоёв.
3. Выбрать элемент и увидеть impact на 1-2-3 шага.
4. Увидеть affected диаграммы.
5. Увидеть orphan elements.
6. Экспортировать подграф в GraphML.

Если этот сценарий работает end-to-end — MVP готов.

---

## OUT OF SCOPE для всего проекта на текущем этапе

1. Редактирование ArchiMate-модели.
2. Создание новых элементов или связей.
3. Запись обратно в Architeezy.
4. Собственная система управления пользователями.
5. Работа с не-ArchiMate моделями (BPMN, UML и т.д.).
6. Интеграция с CI/CD пайплайнами.
7. Real-time синхронизация с Architeezy.
8. AI/ML-анализ архитектуры.
9. Генерация рекомендаций по улучшению модели.
10. Интеграция с ITSM/CMDB системами.
