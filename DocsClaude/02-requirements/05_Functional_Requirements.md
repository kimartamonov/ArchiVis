# 05. Functional Requirements

> Главный вопрос документа: «Какие функции система обязана поддерживать?»

---

## Приоритизация

- **P0** — обязательно для MVP. Без этого MVP нельзя считать завершённым.
- **P1** — желательно для MVP. Включаем, если хватает времени.
- **P2** — post-MVP.

---

## Дополнительные системные поведения

| ID | Требование | P |
|----|-----------|---|
| FR-8.1 | Система работает с одним подключением к Architeezy за раз | P0 |
| FR-8.2 | URL последнего подключения сохраняется в localStorage | P1 |
| FR-8.3 | Token хранится в sessionStorage (очищается при закрытии вкладки) | P0 |
| FR-8.4 | Метрики рассчитываются автоматически при построении графа | P0 |
| FR-8.5 | Битые ссылки в данных: warning в UI, игнорирование при анализе | P0 |
| FR-8.6 | Циклы в графе не вызывают бесконечный цикл (visited set в BFS) | P0 |
