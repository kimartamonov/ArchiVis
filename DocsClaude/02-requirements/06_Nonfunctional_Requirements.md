# 06. Nonfunctional Requirements

> Главный вопрос документа: «Какими качествами должна обладать система?»

---

## 1. Производительность

### Целевой размер модели

| Параметр | Комфортный | Допустимый | Не гарантирован |
|----------|-----------|-----------|-----------------|
| Элементы | до 200 | 200-500 | > 500 |
| Отношения | до 600 | 600-1500 | > 1500 |
| Диаграммы | до 20 | 20-50 | > 50 |

### Целевые времена отклика

| Операция | Целевое время |
|----------|---------------|
| Загрузка модели (до 200 элементов) | < 3 сек |
| Загрузка модели (200-500 элементов) | < 8 сек |
| Impact analysis после выбора элемента | < 1 сек |
| Текстовый поиск (keystroke → результаты) | < 200 мс |
| Применение фильтра | < 500 мс |
| Переключение глубины impact | < 500 мс |
| Pan/zoom графа | 60 fps |
| Переход между экранами | < 300 мс |

Прогресс-бар отображается для операций > 2 секунд.

## 3. Надёжность и устойчивость

| Ситуация | Поведение |
|----------|-----------|
| API недоступен при подключении | Ошибка + кнопка Retry |
| API отдаёт частичные данные | Warning. Граф строится из доступных данных |
| Timeout при загрузке модели | Ошибка после 30 сек + Retry |
| Ошибка 401/403 | «Authentication failed» + redirect на экран подключения |
| Потеря сети во время работы | Работа продолжается на загруженных данных. При перезагрузке — ошибка |

**Ключевой принцип:** после загрузки модели ArchiLens работает полностью offline. Все данные в памяти браузера. Повторные обращения к API не требуются.

---

## 4. Данные и кэширование

| Аспект | Решение |
|--------|---------|
| Хранение данных | In-memory only (Zustand stores) |
| Persistence | Нет. При закрытии вкладки данные теряются |
| Кэш между сессиями | Нет в MVP |
| Что сохраняется в localStorage | URL последнего подключения, ID последней модели |
| Token | sessionStorage (очищается при закрытии вкладки) |

---

## 5. Безопасность

| Аспект | Решение |
|--------|---------|
| Аутентификация | Credentials/token Architeezy. ArchiLens не имеет собственной auth |
| Хранение токена | sessionStorage, не localStorage |
| Передача данных | HTTPS обязателен для подключения к Architeezy |
| XSS защита | Стандартная React-санитизация. Данные из API не вставляются как innerHTML |
| Конфиденциальность | Данные модели не передаются куда-либо. Всё обрабатывается локально в браузере |
| CORS | Architeezy должен разрешать CORS, либо пользователь запускает ArchiLens на localhost |

**Критично:** ArchiLens не передаёт данные модели во внешние сервисы. Никакой telemetry или analytics. Это фундаментальное требование для enterprise-пользователей.

## 6. Объяснимость метрик

Каждая метрика, отображаемая в UI, должна:

1. Иметь **краткое описание** (tooltip) — что означает метрика.
2. Иметь **формулу** (доступна в help/docs) — как она считается.
3. **Не использовать «магические» нормализации** без объяснения.

Пример: `degree = 8` → tooltip: «Total number of direct relationships (incoming + outgoing)».

Это выделено как отдельное NFR, поскольку доверие архитекторов к метрикам — критическое условие принятия продукта.

## 7. Доступность (Accessibility)

Минимальный уровень для MVP:
- Контрастные цвета слоёв (различимы при цветовой слепоте).
- Таблицы навигируемы с клавиатуры.
- Alt-text для ключевых элементов UI.
- Таблица + CSV export — альтернативный путь для тех, кто не может работать с графом.

Полноценный WCAG 2.1 AA — post-MVP.

---

## 8. Расширяемость

Архитектура MVP обеспечивает точки расширения через TypeScript интерфейсы:

1. **Connector interface** — абстракция источника данных.
2. **Metric interface** — абстракция для метрики (имя, описание, формула).
3. **Export interface** — абстракция для экспортёра.

Runtime plugin system (динамическая загрузка модулей) — post-MVP.

## 9. Сопровождаемость

| Аспект | Требование |
|--------|-----------|
| Тесты | Unit > 70% для engine, > 50% для UI. 1 E2E для канонического сценария |
| Модульность | connector / graph-engine / insight-engine / UI — отдельные модули |
| Документация | README с инструкцией запуска. JSDoc для публичных API |
| Кодстайл | ESLint + Prettier. Pre-commit hook |
| API-стабильность | Не гарантирована в MVP. Public API помечается как `@beta` |

## 11. Логирование и диагностика

- Console.log для debug (включается через dev mode).
- Error boundary в React — перехват ошибок рендеринга.
- Счётчик warnings при загрузке (broken references, unknown types) — в UI.
- Нет внешнего logging service в MVP.
